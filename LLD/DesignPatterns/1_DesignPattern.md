## Design Pattern
A design pattern is a reusable, proven solution to common software design problems. It is not code, but a template for structuring code effectively.

## Types of Design Patterns
**1. Creational Design Patterns**
Focus on how objects are created in a controlled and flexible way. They help hide instantiation logic and reduce tight coupling.

**2. Structural Design Patterns**
Focus on how classes and objects are composed/structured. They help form larger structures while keeping them flexible and efficient.

**3. Behavioral Design Patterns**
Focus on how objects communicate and interact with each other. They define responsibility distribution and communication flow between objects.
