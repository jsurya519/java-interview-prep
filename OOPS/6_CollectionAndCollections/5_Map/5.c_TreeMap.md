## TreeMap:
1.  It stores key-value pairs in a sorted order using either a natural or custom comparator.
2.  TreeMap internally uses a Red-Black Tree for efficient sorting.
3.  Provides O(log n) time for insertion, deletion and lookup due to its self-balancing Red-Black Tree structure.
4.  TreeMap does not allow null keys, but allows null values.

## Constructors of TreeMap:

**1. TreeMap()**
This constructor is used to build an empty TreeMap that will be sorted by using the natural order of its keys.
```java
TreeMap<K, V> map = new TreeMap<>();
```

**2. TreeMap(Comparator comp)**
This constructor is used to build an empty TreeMap object in which the elements will need an external specification of the sorting order.
```java
TreeMap<K, V> map = new TreeMap<>(Comparator<? super K> comparator);
```

**3. TreeMap(Map M)**
This constructor is used to initialize a TreeMap with the entries from the given map M which will be sorted by using the natural order of the keys.
```java
TreeMap<K, V> map = new TreeMap<>(Map<? extends K, ? extends V> m);
```

---

```java
public class Dummy {

    public static void main(String[] args){

        Map<Integer, String> mp1 = new TreeMap<>();

        // put(key, value)
        mp1.put(101, "CSE");
        mp1.put(103, "PRD");
        mp1.put(102, "IT");

        System.out.println(mp1);

        Map<Integer, String> mp2 = new TreeMap<>((a,b)->a<b?1:a>b?-1:0);

        // put(key, value)
        mp2.put(101, "CSE");
        mp2.put(103, "PRD");
        mp2.put(102, "IT");

        System.out.println(mp2);
    }
}
// Output:
// {101=CSE, 102=IT, 103=PRD}
// {103=PRD, 102=IT, 101=CSE}
```
