## HashSet:
1. HashSet does not allow duplicate elements.
2. HashSet doesn't maintain any specific order of elements.
3. Uses HashMap internally which is an implementation of hash table data structure.
4. Also implements Serializable and Cloneable interfaces.
5. HashSet is not thread-safe. To make it thread-safe, synchronization is needed externally.
6. Does not support primitive types directly; requires wrapper classes (Integer, Character, etc.).

## Internal Implementation of HashSet:

**Capacity of HashSet:**
Capacity refers to the number of buckets in the hash table. The default capacity of a HashSet is 16 and the load factor is 0.75.
When the number of elements exceeds the threshold, the capacity automatically increases.
```java
Threshold = capacity × load factor
new capacity = old capacity × 2
```

**Internal Implementation:**
HashSet internally uses a HashMap, which is backed by a Node[] (bucket array).
Default capacity is 16, so buckets are indexed from 0 to 15.
When an element is added, its hashCode() is calculated and converted to bucket index using:
```java
index = (capacity - 1) & hash
```
Multiple elements in same bucket are stored using linked list (or tree in Java 8+).
When size exceeds capacity × loadFactor (16 × 0.75 = 12), a new larger array is created and existing elements are redistributed to new buckets.
In Java 8+, during resize an element either stays in same index or moves to oldIndex + oldCapacity.

**Collision in HashSet:**
HashSet internally uses a hash table (via HashMap), where elements are stored based on their hash codes. Sometimes, different elements may produce the same hash code, leading to a collision.

**Note:** HashSet handles collisions using chaining (LinkedList) and converts it into a balanced tree (Red-Black Tree) when the number of elements in a bucket exceeds a threshold (Java 8+).

## Constructors of HashSet:

**1. HashSet()**
Creates a new empty HashSet with default capacity (16) and load factor (0.75).
```java
HashSet<String> set = new HashSet<>();
```

**2. HashSet(int initialCapacity)**
Creates an empty HashSet with the specified initial capacity and default load factor (0.75).
```java
HashSet<Type> set = new HashSet<>( initialCapacity);
```

**3. HashSet(int initialCapacity, float loadFactor)**
Creates an empty HashSet with the given initial capacity and load factor.
```java
HashSet<Type> set = new HashSet<>( initialCapacity, loadFactor);
```

**4. HashSet(Collection<? extends E> c)**
Creates a new HashSet containing the elements of the specified collection (removes duplicates automatically).
```java
HashSet<Type> set = new HashSet<>(c);
```

## Performing Various Operations on HashSet:

```java
import java.util.HashSet;

public class HashSetDemo {

    public static void main(String[] args) {

        // Creating HashSet
        HashSet<String> set = new HashSet<>();

        // add()
        set.add("Apple");
        set.add("Banana");
        set.add("Mango");

        System.out.println("HashSet: " + set);

        // contains()
        System.out.println("Contains Mango? " + set.contains("Mango"));

        // remove()
        set.remove("Banana");

        System.out.println("After removing Banana: " + set);

        // iteration
        System.out.println("Iterating elements:");

        for (String fruit : set) {
            System.out.println(fruit);
        }
    }
}
```
