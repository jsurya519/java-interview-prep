## Contract between hashcode and equals methods:
1. If two objects are equal using equals(), then their hashCode() must be same
```java
if a.equals(b) == true
then a.hashCode() == b.hashCode()
```

2. If hashCodes are same, objects MAY or MAY NOT be equal. This situation is called collision.
3. This equals() + hashCode() contract is important for hash-based collections:
HashMap, LinkedHashMap, HashSet, LinkedHashSet, Hashtable

## Why HashMap, LinkedHashMap, HashSet, LinkedHashSet, HashTable Needs Both

**Example for wrong Implementation:**
```java
import java.util.HashSet;
import java.util.Set;

class Student {

    int id;

    Student(int id) {
        this.id = id;
    }

    // Overriding equals()
    @Override
    public boolean equals(Object obj) {

        Student s = (Student) obj;

        return this.id == s.id;
    }

    // hashCode() NOT overridden
}

public class Main {

    public static void main(String[] args) {

        Student s1 = new Student(101);
        Student s2 = new Student(101);

        // equals() returns true
        System.out.println("s1.equals(s2): "
                + s1.equals(s2));

        // Different hashcodes from Object class
        System.out.println("s1 hashCode: "
                + s1.hashCode());

        System.out.println("s2 hashCode: "
                + s2.hashCode());

        // Adding into HashSet
        Set<Student> set = new HashSet<>();

        set.add(s1);
        set.add(s2);

        // Duplicate objects expected?
        System.out.println("Set size: "
                + set.size());
    }
}
// Output:
// s1.equals(s2): true
// s1 hashCode: 468121027
// s2 hashCode: 1804094807
// Set size: 2
```

When equals() is overridden but hashCode() is not, two logically equal objects can generate different hash codes. HashMap and HashSet first use hashCode() to find the bucket, so equal objects may go into different buckets and duplicates can be stored. Therefore, whenever equals() is overridden, hashCode() must also be overridden to maintain the contract.

**Correct Implementation:**

```java
import java.util.HashSet;
import java.util.Objects;
import java.util.Set;

class Student {

    int id;

    Student(int id) {
        this.id = id;
    }

    // Overriding equals()
    @Override
    public boolean equals(Object obj) {

        if (this == obj)
            return true;

        if (obj == null || getClass() != obj.getClass())
            return false;

        Student s = (Student) obj;

        return this.id == s.id;
    }

    // Overriding hashCode()
    @Override
    public int hashCode() {
        return Objects.hash(id);
    }
}

public class Main {

    public static void main(String[] args) {

        Student s1 = new Student(101);
        Student s2 = new Student(101);

        System.out.println("s1.equals(s2): "
                + s1.equals(s2));

        System.out.println("s1 hashCode: "
                + s1.hashCode());

        System.out.println("s2 hashCode: "
                + s2.hashCode());

        Set<Student> set = new HashSet<>();

        set.add(s1);
        set.add(s2);

        System.out.println("Set size: "
                + set.size());
    }
}
// Output:
// s1.equals(s2): true
// s1 hashCode: 132
// s2 hashCode: 132
// Set size: 1
```

Both equals() and hashCode() are overridden correctly.
Since equal objects now produce the same hash code, HashSet places them in the same bucket and prevents duplicates.
